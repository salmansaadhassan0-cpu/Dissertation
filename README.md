# Urban Air Mobility Fleet Management using Discrete-Event Simulation (DES)

This repository contains the MATLAB implementation developed for a fourth-year Mechanical Engineering dissertation investigating fleet management in Urban Air Mobility (UAM) systems.

The project uses a discrete-event simulation (DES) framework to analyse how operational constraints, particularly pilot duty limitations, affect overall system performance.

---

## Project Overview

Three progressively constrained models are implemented:

- Model A — Baseline  
  Random dispatch across a small network. Used to validate the core simulation mechanics.

- Model B — Demand-Driven Network  
  Large-scale network (100 nodes) with demand-weighted routing. Represents an unconstrained upper-bound system.

- Model C — Crew-Constrained System  
  Integrates aircraft and pilot scheduling with duty limitations. Represents a more realistic operational scenario.

---

## Repository Contents

- fleet_sheduling_system_model_A.m  
  Baseline simulation (random dispatch)

- fleet_sheduling_system_model_B.m  
  Demand-driven routing model and shared network generator

- fleet_sheduling_system_model_C.m  
  Crew-constrained simulation using the shared topology

- validation.m  
  Runs eight validation checks on Model C outputs

- run_modelC_unlock_test.m  
  Removes crew constraints to evaluate the unconstrained recovery case

- run_modelC_stability_tests.m  
  Executes Model C across multiple random seeds to assess statistical stability

- radar_comparison_between_routes.m  
  Generates comparison figures between Model B and Model C

7. Run radar_comparison_between_routes.m  

---

## Validation Approach

Model C is validated using the following checks:

1. Aircraft non-overlap  
2. Pilot non-overlap  
3. Daily duty compliance  
4. Operating window compliance  
5. Departure capacity constraints  
6. Route feasibility  
7. Full-unlock sensitivity test  
8. Multi-run statistical stability  

---

## Key Observations

- Demand-driven routing introduces strong hub concentration effects (Model B)
- Adding crew constraints leads to a system-wide reduction in performance (Model C)
- Network coverage remains relatively stable, while demand satisfaction drops significantly
- The dominant limitation arises from operational synchronisation rather than network structure

---

## Notes

- Model C depends on the shared network generated by Model B
- Validation checks 7 and 8 require the unlock and stability test outputs
- Results are stochastic but remain consistent across different seeds
